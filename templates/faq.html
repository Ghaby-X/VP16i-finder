Q1. What is Herpes-finder?

Herpes-finder is a webserver that is capable of predicting the inhibitory activity of compounds in their smile notation against herpes simplex virus viral protein 16

Q2. How does Herpes-finder work?

It works as follows

  • It receives either a single or batch query input alongside the type of model to use from the user via the input interface
  • In server logic, it computes morgan fingerprints from the input smile notation and feed them to pre-trained machine learning models for predicting inhibitors of herpes
  • It displays the activity and confidence of each prediction

See tutorial section to know how to use Herpes-finder

Q3. What does it mean for a compound to be within AD?

Applicability domain defines the region or chemical space for which our model predictions is reliable. A compound in the applicability domain means, the compound is similar to our training compounds thus the model prediction is reliable

Q4. Why is my list of molecules prediction throwing an Error?

If prediction for batch compounds show an error, check the following ?

  • Ensure the csv file do not contain any headers or title
  • Ensure the smiles present in the csv are valid
  • Ensure the smiles are arranged in a single column
  • Ensure the smiles are continuous. i.e., there should be no empty entry between two smiles input

Q5. Can I download prediction results from Herpes-finder?

Yes, you can download the prediction results. To download, click on the download button at the top-left of the results and the download should begin

Q6. Will there be any feature updates or modifications to Herpes-finder?

Yes, we would like to enhance the capabilities of Herpes-finder predictions by retraining model with more dataset.
